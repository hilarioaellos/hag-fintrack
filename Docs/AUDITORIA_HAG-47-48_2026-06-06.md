# Auditoría HAG-47 + HAG-48 — Notifications i18n

**Fecha:** 2026-06-06  
**Issues:** HAG-47, HAG-48  
**Módulo:** Notifications (backend + frontend)

---

## Problemas

### HAG-47 — Mensaje almacenado en inglés en la BD
`checkPaymentDueDates` (internalMutation) generaba:
```
message: `Payment due today for ${account.name}`
```
Este string en inglés se almacenaba en la BD. El cron corre server-side y no conoce el locale del usuario.

### HAG-48 — aria-label hardcodeado
`NotificationBell.tsx:23`: `aria-label="Notifications"` — siempre en inglés.

### Adicional (sin issue separado)
`{n.severity}` mostraba "urgent"/"warning"/"info" en inglés como badge.

---

## Solución — HAG-47

**Patrón:** backend almacena solo el dato dinámico; frontend renderiza el mensaje en el idioma activo usando el campo `type`.

- **Backend:** `message: account.name` (solo el nombre de cuenta — el dato necesario)
- **Frontend:** `n.type === "payment_due" ? t("paymentDue", { account: n.message }) : n.message`
- **Fallback:** si `type` es desconocido, muestra `n.message` crudo (backward compatible con notificaciones existentes en inglés)

---

## Solución — HAG-48 + severity

- `aria-label="Notifications"` → `aria-label={t("ariaLabel")}`
- `{n.severity}` → `{t(\`severity.${n.severity}\`)}`

---

## Claves i18n añadidas

```json
"notifications": {
  "ariaLabel": "Notifications" / "Notificaciones",
  "paymentDue": "Payment due today for {account}" / "Pago vence hoy para {account}",
  "severity": {
    "urgent": "Urgent" / "Urgente",
    "warning": "Warning" / "Advertencia",
    "info": "Info" / "Info"
  }
}
```
