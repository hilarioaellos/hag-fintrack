## Fases hacia producción

### Fase 2 — Usuarios reales en dev ✅ Iniciada 2026-06-06

- QA completo ✅
- Datos de prueba eliminados con `clearUserData` (Settings → Zona de Riesgo) ✅
- Invitar 2 usuarios vía super admin en hagpartner.com 🔲
- Observar 1-2 semanas con uso real 🔲

**Flujo de invitación:** Admin crea invitación en hagpartner.com → usuario recibe email → crea contraseña → entra a fintrack.hagpartner.com con las mismas credenciales. Cada usuario ve solo sus propios datos (`userId` en todas las queries).

**Nota operacional:** Cambios en el backend Convex requieren `npx convex dev --once` en `Landingpage-HAG-Partner/` para desplegarse a `focused-swan-416`. El git push solo actualiza el código fuente, no el runtime de Convex.

### Fase 3 — Migración a Convex prod (wandering-herring-170)

Pre-checklist:
- [ ] `NEXT_PUBLIC_CONVEX_URL` en Railway hag-fintrack → `https://wandering-herring-170.convex.cloud`
- [ ] Auth callback URLs configuradas en Convex prod
- [ ] `NEXT_PUBLIC_FINTRACK_URL` en Railway Landingpage-HAG-Partner no cambia

Pasos:
1. `npx convex deploy` en Landingpage-HAG-Partner → publica a `wandering-herring-170`
2. Actualizar `NEXT_PUBLIC_CONVEX_URL` en Railway hag-fintrack → redeploy

4. Re-ingresar datos (Convex no migra entre deployments)
